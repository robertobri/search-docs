SearchGrid
Root response object — all fields
root object
Top-level object returned by the search grid endpoint. Contains 3 required fields (sections, promotionalBanners, context) plus several optional fields that enrich the browsing experience.
FieldTypeReq.ConstraintsDescription
id string optional UUID v4 Unique identifier of the grid definition. Only present on IOP grids.
e.g. f7eda7de-9352-490f-978d-c41c0b188c87
milestoneCode string optional 1–256 chars Code of the milestone associated to this grid.
e.g. night-propagation-20-09-2023
sections* [GridSection] required — Array of sections that compose the grid. Each section groups content blocks under a semantic type. See GridSection.
promotionalBanners* [PromotionalBanner] required sorted by priority Promotional banners configured for this grid/store combination, sorted by descending priority. See PromotionalBanner.
topbar MenuItemTopBar optional maxLength: 100000 Top navigation bar with browsable categories within the grid. Contains header and items*.
context* GridContext required — Additional context required to render the grid correctly. See GridContext.
style GridStyle optional — Unified styling configuration: backgroundColor, tintColor, font, layout, margin, behaviour.
filters [SearchFilter] optional maxItems: 30 · default: [] Available filters to refine the search results.
sortings SearchGridSortings optional — Available sorting options for the grid.
section string optional maxLength: 20 Section of the response.
WOMANMANKIDZ3DHOMEMASSIMODUTTI-WOMANMASSIMODUTTI-MAN
metadata SearchGridMetadata opcional — Analytics metadata: provider, searchpass, spellchecked, queryAnalysis.
additionalEntities [SearchAdditionalEntity] optional maxItems: 200 · default: [] Additional (non-product) entities returned alongside the grid. May include editorial content, banners, etc.
GridSection
Item of sections[] — groups blocks with a shared semantic type
sections[]
A section is the main structural unit inside a grid. The type field drives visual and functional treatment. The elements field holds the renderable blocks. All attribute arrays must be present even if empty.
FieldTypeReq.ConstraintsDescription
id string optional UUID Section identifier.
e.g. 77f97b3c-7da5-4d82-8485-d6ea216234fe
type* string required — Differentiates sections with specific features. main is the default value.
main stock-clearance product-recommendations novelty default-search
The list is non-exhaustive; additional values may exist for specific use cases.
title string optional — Section display title. If both this field and header are present, display the header content instead.
e.g. Ultimas Tallas
elements* [GridBlock] required — Content blocks within the section. Each block is independently renderable. See GridBlock.
attributes* [ValuedAttribute] required — Section configuration attributes. See ValuedAttribute. Common key: EDITORIAL_SECTION: true.
autoTemplates [AutoTemplate] optional — Auto-template info for the section. Only present when the corresponding setter is active. Contains name*, numberOfElements, minElements, maxElements, autoBlocks, fallbacks.
header Header optional — Rich section header. Takes visual precedence over title when both are present.
blocksBeforePersonalization array optional — Blocks that must be rendered before the personalization block within the section.
GridBlock
Item of sections[].elements[] — atomic renderable unit of the grid
elements[]
The block is the fundamental rendering unit. The kind field determines which subtype populates metadata. The linkedBlocks and tags fields are always required regardless of the block type.
FieldTypeReq.ConstraintsDescription
id* string required GridBlockId Unique block identifier (UUID). Referenced in linkedBlocks of other blocks.
e.g. 9e0eafe2-2ba5-468a-a49a-28fc3dd39254
name string optional — Descriptive block name for debugging and traceability.
e.g. WOMAN-SHIRTS-V2022
kind* string required enum Block type. Determines the shape of metadata.
EDITORIAL AUTOTEMPLATE SDUI CAROUSEL PERSONALIZATION
isSectionIndex* boolean required — Whether this block acts as a section index. When true, the frontend should treat it as a navigation anchor within the grid.
e.g. false
metadata* object (anyOf) required anyOf by kind Block-specific configuration and content. Internal shape varies by kind:
EDITORIAL → EditorialGridBlock
AUTOTEMPLATE → AutotemplateGridBlock · contains commercialComponents*
SDUI → SDUIGridBlock
CAROUSEL → CarouselGridBlock
PERSONALIZATION → PersonalizationGridBlock
style GridStyle optional — Block styles: backgroundColor, layout (e.g. 4C), template (SDUI only, e.g. 1A), margin, behaviour.
linkedBlocks* [GridBlockId] required minItems: 2 List of linked blocks (including this one) that must always be rendered consecutively. The frontend must not place other elements between blocks belonging to the same linked group. Minimum 2 items (block includes itself).
tags* [string] required — Block tags for classification and behaviour. Can be used for conditional rendering logic.
e.g. ["HOME"]
attributes array optional — Additional block attributes in key-value format.
Note on linkedBlocks: The frontend must ensure all blocks whose IDs appear in linkedBlocks are rendered in adjacent positions. Breaking this consecutiveness produces incorrect layouts.
AutotemplateGridBlock — commercialComponents
Shape of SimplifiedCommercialComponentGrid — the product object
metadata.commercialComponents[]
When kind = AUTOTEMPLATE, the field metadata.commercialComponents* is required and holds the block's products. The key fields of each commercial component for frontend rendering are documented below.
FieldTypeDescription
identifier object Uniquely identifies the commercial component.
id — Internal numeric ID e.g. 16546465
productParentId — Parent product ID e.g. 223703785
reference — Full reference e.g. C07380704800000-V2023
productParentReference — Parent reference e.g. 07380704-V2023
displayReference — Display reference e.g. 7380/704
name string Product display name for UI.
e.g. COMFORT PERFORMANCE TECHNICAL BLAZER
type string Commercial component type. e.g. Product
state object Product state. Key field: visibility — controls whether the product is visible in UI.
e.g. { "visibility": "visible" }
classification object Catalogue classification: brand (id, groupId, groupCode), section, family, subfamily, sizeSystem.
color object Primary color and alternative colors of the component.
id — Color ID
name — Color name e.g. Black
hexCode — Color hex e.g. #000000
otherColors[] — Available color variants (id, reference, name, hexCode, colorcutUrl)
pricing object Product pricing structure. Monetary values are expressed in smallest currency unit (divide by 10^|exponent|).
price.current — Current price: value, minValue, currency (code, exponent)
originalPrice — Pre-discount price + discount.label
futurePrice — Upcoming price with promotion info (promotion.name, startDate, endDate)
unitPrice — Price per unit of measure (unit, type)
media array Product media assets (images, videos). Each item contains:
asset.type — Type: image / video
asset.url — Base asset URL
asset.dimensions — Dimensions: height, width
asset.styles — Asset presentation styles
asset.editorialName — Editorial name e.g. a1
asset.locations — Where to use the asset: CATEGORY_PAGE, PRODUCT_PAGE
layers[] — Additional layers on top of the main asset
sizes array Available sizes. Each size includes:
identifier — id, reference, sku, equivalentSizeId
nomenclature — name, shortName, description
availability — in_stock / out_of_stock
sizeType — regular / others
pricing — Size-specific price (may differ from product price)
skuDimensions[] — Physical dimensions (HEIGHT, WIDTH, DEPTH)
twinnedSkus[] — Linked twinned SKUs
tags array Product tags for UI badges. Each tag contains displayName and type.
e.g. { "displayName": "NEW", "type": "NEW" }
features object Product feature flags. Relevant keys for frontends:
shouldUseColorcutInColorLabel — Use colorcut in the color selector
isSizeLabelExplanationRequired — Show size tooltip
highlightPrice — Visually highlight the price
isVirtualTryOnSupported — Enable Virtual Try-On
seo object Product SEO metadata: keyword, seoProductId, discernProductId, irrelevant (if true do not index).
firstAvailabilityDate string First product availability date in ISO 8601 format.
e.g. 2024-10-03T23:00:00Z
Price calculation: Monetary values use currency.exponent to determine the scale. With exponent: -2 and value: 2999 the price is 29.99 EUR. Compute as: value * 10^exponent.
Size-level pricing: Each size may carry its own pricing object that differs from the product-level price. Always use the size price when available.

PromotionalBanner
Item of promotionalBanners[] — grid promotional banners
promotionalBanners[]
Banners sorted by descending priority. The first three fields are required. The frontend must render banners respecting the array order.
FieldTypeReq.ConstraintsDescription
id* integer required — Unique numeric banner identifier. Use for deduplication and as list key.
e.g. 1
title* string required — Main banner text, internationalised. Display in the component heading.
e.g. NEW IN
description* string required — Descriptive banner text to display below the title.
e.g. Descubre la nueva coleccion
disableClose boolean optional — When true, the frontend must remove the close button from the banner. The user will not be able to dismiss it.
e.g. true
link PromotionalBannerLink optional — Navigation target on banner click. When present, the whole component must be clickable.
GridContext
Field context* of SearchGrid — rendering configuration
context
Required object holding additional information needed for correct grid and product rendering. The three required fields must be processed by the frontend before rendering any block.
FieldTypeReq.Description
isPlainImageSelectorEnabled* — required Controls whether the plain image selector is enabled for this grid. Affects the color selection experience in product components.
isOverlayEnabled — optional Whether the overlay is enabled for the grid (e.g. quick-add, image zoom).
gridPaginationInfo GridPaginationInfo optional Pagination info: total elements, current page, page size. Required to implement infinite scroll or pagination.
autoTemplatePattern* — required Auto-template distribution pattern applied to the grid. Defines how product blocks are laid out visually in the grid.
attributes* [ValuedAttribute] required Context attributes in key-value format that configure the global rendering behaviour. See ValuedAttribute.
categoryTitle — optional Category title to display in the grid page heading.
autoTemplates array optional Definition of the available auto-templates for the grid.
ValuedAttribute
Dynamic key-value attribute — used in sections[].attributes and context.attributes
ValuedAttribute
Generic key-value attribute structure used at multiple points in the schema. Both fields are required. Values are always string even when representing booleans or numbers — the frontend must parse them according to the key's semantics.
FieldTypeReq.ConstraintsDescription
key* string required — Attribute key in SCREAMING_SNAKE_CASE format.
e.g. EDITORIAL_SECTION
value* string required — Attribute value serialised as a string. Parse according to the key's semantics.
e.g. "true"
Integration tip: Boolean values arrive as strings "true" / "false". Use strict string comparison or parse with value === "true" to avoid JavaScript coercion errors.
