SearchGrid
Root response object — all fields
root object
Top-level object returned by the search grid endpoint. Contains sections (the product grid), context (pagination and rendering config), plus filters, sortings and metadata.
FieldTypeReq.ConstraintsDescription
sections* [GridSection] required — Array of sections that compose the grid. In this API each section has type (and matching id when sent) of default-search or alternatives-search. See GridSection.
context* GridContext required — Additional context required to render the grid correctly. See GridContext.
filters [SearchFilter] optional maxItems: 30 · default: [] Available filters to refine the search results.
sortings SearchGridSortings optional — Available sorting options for the grid.
section string optional maxLength: 20 Section of the response.
WOMANMANKIDZ3DHOMEMASSIMODUTTI-WOMANMASSIMODUTTI-MAN
metadata SearchGridMetadata opcional — Analytics metadata: provider, searchpass, spellchecked, queryAnalysis.
additionalEntities [SearchAdditionalEntity] optional maxItems: 200 · default: [] Additional (non-product) entities returned alongside the grid. May include redirects, banners, etc.
GridSection
Item of sections[] — groups blocks with a shared semantic type
sections[]
A section is the main structural unit inside a grid. The type field drives visual and functional treatment. The elements field holds the renderable blocks. All attribute arrays must be present even if empty.
Search API: section id and type always carry the same value, and only two exist: default-search and alternatives-search. For alternatives-search, when building a single “similars” experience, consume similars-results first, then personalization-results if present; the service controls counts and may omit personalization (e.g. initial production: similars with a cap and other colours, without personalization blocks).
FieldTypeReq.ConstraintsDescription
id string optional default-search | alternatives-search When present, must equal type (same two values as the section discriminator).
e.g. default-search · alternatives-search
type* string required default-search | alternatives-search Only these two section types exist in this API (and id equals type when sent):
default-search alternatives-search
default-search — main search results grid, same treatment regardless of which search provider produced the response.
alternatives-search — alternative results (other colours, similars, optional personalization from search cross-sell).
title string optional — Section display title. If both this field and header are present, display the header content instead.
e.g. Ultimas Tallas
elements* [GridBlock] required — Content blocks within the section. Each block is independently renderable. See GridBlock.
When type is default-search, you may receive multiple blocks whose id is search-results — render them without distinguishing by provider. When type is alternatives-search, block id values identify alternative rails (see GridBlock id).
attributes* [ValuedAttribute] required — no relevant for search at the moment
autoTemplates [AutoTemplate] optional — Auto-template info for the section. Only present when the corresponding setter is active. Contains name*, numberOfElements, minElements, maxElements, autoBlocks, fallbacks.
header Header optional — Rich section header. Takes visual precedence over title when both are present.
blocksBeforePersonalization array optional — Blocks that must be rendered before the personalization block within the section.
GridBlock
Item of sections[].elements[] — atomic renderable unit of the grid
elements[]
The block is the fundamental rendering unit. The kind field determines which subtype populates metadata. The linkedBlocks and tags fields are always required regardless of the block type.
FieldTypeReq.ConstraintsDescription
id* string required GridBlockId Unique block identifier within the response. May be a UUID (referenced in linkedBlocks) or a stable string slug in Search API payloads.
Search — under default-search: blocks with id search-results (render all the same).
Search — under alternatives-search: known provider ids: other-colors-results (remaining MOCACO colours), similars-results (similar products), personalization-results (search cross-sell grid; use after similars to fill the similars rail when sent). The service may omit providers (e.g. no personalization).
e.g. 9e0eafe2-2ba5-468a-a49a-28fc3dd39254 · search-results
name string optional — Descriptive block name for debugging and traceability.
e.g. WOMAN-SHIRTS-V2022
kind* string required enum no relevant for search at the moment
isSectionIndex* boolean required — no relevant for search at the moment
metadata* object (anyOf) required anyOf by kind Block-specific configuration and content. Internal shape varies by kind:
EDITORIAL → EditorialGridBlock
AUTOTEMPLATE → AutotemplateGridBlock · contains commercialComponents*
SDUI → SDUIGridBlock
CAROUSEL → CarouselGridBlock
PERSONALIZATION → PersonalizationGridBlock
style GridStyle optional — Block styles: backgroundColor, layout (e.g. 4C), template (SDUI only, e.g. 1A), margin, behaviour.
linkedBlocks* [GridBlockId] required minItems: 2 no relevant for search at the moment
tags* [string] required — no relevant for search at the moment
attributes array optional — Additional block attributes in key-value format.
AutotemplateGridBlock — commercialComponents
Shape of SimplifiedCommercialComponentGrid — the product object
metadata.commercialComponents[]
When kind = AUTOTEMPLATE, the field metadata.commercialComponents* is required and holds the block's products. The key fields of each commercial component for frontend rendering are documented below.
FieldTypeDescription
identifier object Product identifiers and references. Relevant keys for frontends:
id
productParentId
reference
productParentReference
displayReference
name string Product display name for UI.
e.g. COMFORT PERFORMANCE TECHNICAL BLAZER
type string Commercial component type. e.g. Product
state object Product state. Key field: visibility — controls whether the product is visible in UI.
e.g. { "visibility": "visible" }
classification object Catalogue classification: brand (id, groupId, groupCode), section, family, subfamily, sizeSystem.
color object Primary color and alternative colors of the component.
id — Color ID
name — Color name e.g. Black
hexCode — Color hex e.g. #000000
otherColors[] — Available color variants (id, reference, name, hexCode, colorcutUrl)
pricing object Product pricing structure. Monetary values are expressed in smallest currency unit (divide by 10^|exponent|).
price.current — Current price: value, minValue, currency (code, exponent)
originalPrice — Pre-discount price + discount.label
futurePrice — Upcoming price with promotion info (promotion.name, startDate, endDate)
unitPrice — Price per unit of measure (unit, type)

asset.type

tags array Product tags for UI badges. Each tag contains displayName and type.
e.g. { "displayName": "NEW", "type": "NEW" }

shouldUseColorcutInColorLabel

keyword

firstAvailabilityDate string First product availability date in ISO 8601 format.
e.g. 2024-10-03T23:00:00Z
Price calculation: Monetary values use currency.exponent to determine the scale. With exponent: -2 and value: 2999 the price is 29.99 EUR. Compute as: value * 10^exponent.
Size-level pricing: Each size may carry its own pricing object that differs from the product-level price. Always use the size price when available.

PromotionalBanner
Item of promotionalBanners[] — grid promotional banners
promotionalBanners[]
Banners sorted by descending priority. The first three fields are required. The frontend must render banners respecting the array order.
FieldTypeReq.ConstraintsDescription
id* integer required — Unique numeric banner identifier. Use for deduplication and as list key.
e.g. 1
title* string required — Main banner text, internationalised. Display in the component heading.
e.g. NEW IN
description* string required — Descriptive banner text to display below the title.
e.g. Descubre la nueva coleccion
disableClose boolean optional — When true, the frontend must remove the close button from the banner. The user will not be able to dismiss it.
e.g. true
link PromotionalBannerLink optional — Navigation target on banner click. When present, the whole component must be clickable.
GridContext

Object holding additional information needed for correct grid rendering. Contiene la paginación, el patrón de layout y los atributos globales de configuración.

categoryTitle — optional Category title to display in the grid page heading.
